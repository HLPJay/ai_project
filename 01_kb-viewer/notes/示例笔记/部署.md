  # 还没有域名，如何安装证书呢？

  如果还没有域名，配置 HTTPS 证书有以下几种方案：

  方案1：使用 Cloudflare Tunnel（推荐，最简单）

  Cloudflare Tunnel 可以在没有域名和公网 IP 的情况下，安全地暴露本地服务，并自动获得 HTTPS。

  安装步骤：

  # 1. 安装 cloudflared
  # Ubuntu/Debian
  wget -q https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
  sudo dpkg -i cloudflared-linux-amd64.deb

  # 2. 登录 Cloudflare（需要免费账号）
  cloudflared tunnel login

  # 3. 创建隧道
  cloudflared tunnel create kb-viewer

  # 4. 配置路由（创建时会显示隧道ID）
  cloudflared tunnel route dns <隧道ID> kb-viewer.your-name.cloudflaretunnel.com

  # 5. 创建配置文件
  cat > ~/.cloudflared/config.yml << EOF
  tunnel: <隧道ID>
  credentials-file: /root/.cloudflared/<隧道ID>.json

  ingress:
    - hostname: kb-viewer.your-name.cloudflaretunnel.com
      service: http://localhost:3000
    - service: http_status:404
  EOF

  # 6. 启动隧道
  cloudflared tunnel run <隧道ID>

  # 7. 设为系统服务（可选）
  sudo cloudflared service install
  sudo systemctl start cloudflared

  优点：
  - 完全免费
  - 无需域名、无需端口转发
  - 自动 HTTPS
  - Cloudflare 的 CDN 和安全保护

  访问地址：
  https://kb-viewer.your-name.cloudflaretunnel.com

  方案2：自签名证书（适合内部使用）

  生成自签名证书：

  # 1. 安装 OpenSSL
  sudo apt install -y openssl

  # 2. 生成私钥和证书（有效期365天）
  sudo mkdir -p /etc/ssl/kb-viewer
  cd /etc/ssl/kb-viewer

  sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout kb-viewer.key \
    -out kb-viewer.crt \
    -subj "/C=CN/ST=Beijing/L=Beijing/O=YourOrg/OU=IT/CN=your-server-ip"

  # 3. 设置权限
  sudo chmod 600 kb-viewer.key

  配置 Nginx 使用自签名证书：

  sudo nano /etc/nginx/sites-available/kb-viewer-ssl

  server {
      listen 443 ssl http2;
      server_name your-server-ip;  # 或直接使用 _ 匹配所有

      ssl_certificate /etc/ssl/kb-viewer/kb-viewer.crt;
      ssl_certificate_key /etc/ssl/kb-viewer/kb-viewer.key;

      ssl_protocols TLSv1.2 TLSv1.3;
      ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
      ssl_prefer_server_ciphers off;

      location / {
          proxy_pass http://127.0.0.1:3000;
          proxy_http_version 1.1;
          proxy_set_header Upgrade $http_upgrade;
          proxy_set_header Connection 'upgrade';
          proxy_set_header Host $host;
          proxy_cache_bypass $http_upgrade;
          proxy_set_header X-Real-IP $remote_addr;
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
          proxy_set_header X-Forwarded-Proto $scheme;
      }
  }

  # HTTP 重定向到 HTTPS
  server {
      listen 80;
      server_name your-server-ip;
      return 301 https://$server_name$request_uri;
  }

  浏览器警告处理：
  用户访问时浏览器会显示"不安全"警告，需要手动：
  1. 点击"高级"
  2. 点击"继续前往your-server-ip（不安全）"

  适用场景：
  - 内部团队使用
  - 测试环境
  - 不介意浏览器警告

  方案3：使用免费子域名 + Let's Encrypt

  使用 DuckDNS（免费子域名）：

  # 1. 访问 https://www.duckdns.org 注册账号
  # 2. 获取一个免费子域名，如：your-name.duckdns.org
  # 3. 获取你的 Token

  # 4. 在服务器上安装 DuckDNS 脚本
  sudo apt install -y curl
  sudo mkdir -p /opt/duckdns
  sudo nano /opt/duckdns/duck.sh

  #!/bin/bash
  echo url="https://www.duckdns.org/update?domains=your-name&token=your-token&ip=" | curl -k -K -

  # 5. 设置权限和定时更新
  sudo chmod +x /opt/duckdns/duck.sh
  sudo crontab -e
  # 添加：*/5 * * * * /opt/duckdns/duck.sh >/dev/null 2>&1

  # 6. 手动运行一次
  sudo /opt/duckdns/duck.sh

  使用 Certbot 获取证书：

  # 安装 Certbot
  sudo apt install -y certbot

  # 使用 DNS 验证（不需要 80 端口开放）
  sudo certbot certonly --manual --preferred-challenges=dns \
    -d your-name.duckdns.org \
    --register-unsafely-without-email

  # 按提示在 DuckDNS 添加 TXT 记录验证

  配置 Nginx：

  server {
      listen 443 ssl http2;
      server_name your-name.duckdns.org;

      ssl_certificate /etc/letsencrypt/live/your-name.duckdns.org/fullchain.pem;
      ssl_certificate_key /etc/letsencrypt/live/your-name.duckdns.org/privkey.pem;

      # ... 其他配置同上
  }

  优点：真正的有效证书，无浏览器警告
  缺点：需要每 90 天续期，DNS 更新可能有延迟

为什么必须备份？

  1. 数据价值 > 服务器成本

  - 你的知识库笔记是不可再生的智力资产
  - 服务器月租可能50元，但丢失几年的笔记价值无法计算
  - 一次硬盘故障就会永久丢失所有数据

  2. 服务器风险真实存在

  常见数据丢失场景：
  • 硬盘物理损坏（云盘也有0.1%年故障率）
  • 误操作：rm -rf /opt/kb-viewer/notes/*
  • 软件bug：Node.js应用崩溃时文件损坏
  • 安全攻击：勒索软件加密你的.md文件
  • 云厂商故障：虽然罕见，但阿里云/腾讯云都有过事故

  3. 你的应用架构特点

  - 数据直接存在服务器文件系统，没有数据库的事务保护
  - 编辑器直接修改文件，没有版本控制（如Git）
  - 文件删除通过API直接 fs.unlinkSync()，没有回收站

  ---
  备份策略建议

  层级1：本地自动备份（最基本）

  # 每天凌晨2点备份，保留7天
  0 2 * * * /opt/backup-kb.sh

  层级2：异地备份（重要）

  # 每周同步到另一个存储位置
  # 方案A：同步到另一台服务器
  rsync -avz /opt/kb-viewer/notes/ user@backup-server:/backups/kb-viewer/

  # 方案B：同步到对象存储（推荐）
  # 安装阿里云OSS工具或腾讯云COS工具
  ossutil64 cp -r /opt/kb-viewer/notes/ oss://your-bucket/kb-backup/

  层级3：版本化备份（最佳实践）

  # 使用 restic 或 borg 做增量加密备份
  # 安装 restic
  apt install restic

  # 初始化备份仓库（到OSS或本地）
  restic -r oss:your-bucket/kb-repo init

  # 每天增量备份
  restic -r oss:your-bucket/kb-repo backup /opt/kb-viewer/notes/

  ---
  具体备份什么？

  1. 核心数据：/opt/kb-viewer/notes/ 目录下所有 .md 文件
  2. 配置文件：server.js、crud-api.js（如果自定义过）
  3. PM2 进程列表：pm2 save 导出的应用状态

  不要备份：
  - node_modules/（可通过 package.json 重建）
  - 日志文件（可重新生成）
  - 临时文件

  # 一个遗留小问题
    同样的代码在windows环境上部署启动和linux环境上部署启动，展示效果有差异。
    ===》底层渲染机制不同导致？
    ===》最后发现是edge浏览器默认把网页扩大到120%左右显示导致。


     为什么 Windows 正常，云服务器不正常：
  - 浏览器窗口大小/分辨率不同
  - 缩放比例不同
  - 云服务器可能通过远程桌面或不同分辨率访问

  解决方案

  方案1：为内容区域添加右边距（推荐）

  在 index.html 的 CSS 部分添加以下规则（建议加在第 775-778 行附近）：

  /* TOC sidebar */
  #toc-sidebar {
    /* 现有样式不变 */
  }

  #toc-sidebar.visible {
    transform: translateX(0);
  }

  /* 当 TOC 显示时，为内容区域添加右边距 */
  #toc-sidebar.visible ~ #workspace #main #content {
    padding-right: 240px; /* 与 TOC 宽度相同 */
  }

  /* 确保移动端响应式 */
  @media (max-width: 800px) {
    #toc-sidebar.visible ~ #workspace #main #content {
      padding-right: 200px; /* 移动端 TOC 宽度为 200px */
    }
  }

  方案2：调整 #md-view 的最大宽度

  /* 当 TOC 显示时，减少 md-view 的最大宽度 */
  #toc-sidebar.visible ~ #workspace #main #md-view {
    max-width: calc(840px - 240px);
    margin-right: auto;
    margin-left: auto;
  }

  @media (max-width: 1200px) {
    #toc-sidebar.visible ~ #workspace #main #md-view {
      max-width: calc(90% - 240px);
    }
  }

  @media (max-width: 800px) {
    #toc-sidebar.visible ~ #workspace #main #md-view {
      max-width: calc(100% - 200px);
      padding: 24px 20px;
    }
  }

  方案3：使用 CSS Grid 重新定义布局（更彻底）

  在 #workspace 上使用 CSS Grid：

  #workspace {
    display: grid;
    grid-template-columns: var(--sw) 1fr;
    grid-template-areas: "sidebar main";
    overflow: hidden;
    position: relative; /* 为 fixed TOC 提供参考 */
  }

  #sidebar {
    grid-area: sidebar;
  }

  #main {
    grid-area: main;
    display: flex;
    flex-direction: column;
    position: relative;
  }

  /* TOC 显示时调整 main 的右边距 */
  #toc-sidebar.visible ~ #workspace #main {
    margin-right: 240px;
    transition: margin-right 0.25s ease;
  }
