## 决策 001: 技术栈选择
- **日期**: 2026-04-01
- **决策**: C++17 + Qt6 (QWidgets) + OpenCV 4.x
- **原因**: 开发者 C++ 背景，Qt6 提供原生级窗口管理和屏幕捕获能力，OpenCV C++ 接口是图像处理第一公民
- **放弃的方案**: Electron（性能差、体积大）、Tauri+Rust（学习成本高、图像生态弱）、Python+PySide6（有 Qt 不必绕 Python）
- **影响范围**: 全局

## 决策 002: Qt 许可选择
- **日期**: 2026-04-01
- **决策**: LGPL，动态链接 Qt 库
- **原因**: 开源项目，LGPL 允许免费使用，用户可替换 Qt 库
- **约束**: 所有 Qt 模块必须动态链接，不能静态编译 Qt
- **影响范围**: 构建配置、打包
