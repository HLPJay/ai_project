# 架构对比与总结

## 一、重构前后对比

| 维度 | 重构前 (Chained Scripts) | 重构后 (Managed Workflow) |
|------|--------------------------|---------------------------|
| **执行逻辑** | A -> B -> C (死板) | 任务调度器根据状态触发 (灵活) |
| **容错性** | 一处断电，全盘重来 | 支持断点续传、异常重试 |
| **扩展性** | 修改一个脚本可能影响全局 | 插件式替换，互不干扰 |
| **数据流** | 物理文件散落在各处 | 统一的 Context 内存映射 + 结构化存储 |
| **状态追踪** | 无 | metadata.json 全生命周期记录 |
| **异常处理** | 简单退出 | 熔断机制 + 错误日志 |
| **网络依赖** | 硬编码代理 | 环境变量统一透传 |
| **日志系统** | 散乱 print | llm_logger.py 单例模式 |

---

## 二、核心架构优化点

### 1. 断点续传能力
```
优化前：图片生成到第50张断电 → 重新生成全部（浪费 API 费用）
优化后：检测 scene_049.png 已存在 → 直接跳过，从第51张继续
```

### 2. 环境隔离
```
优化前：analyze_srt.py 硬编码 127.0.0.1:7890
优化后：所有 Python 脚本直接使用 os.environ.get("HTTP_PROXY")，代理在 config.sh 中统一配置
```

### 3. 状态可视化
```
优化前：运行报错很难定位
优化后：metadata.json 记录每个 Skill 的 status/output/error
```

| llm_logger.py | Python | 单例日志系统 |

## 四、三层架构总结

```
┌─────────────────────────────────────────────────────────────┐
│                    Layer 3: 技能层 (Skills)                  │
│  generate_lyrics.py → generate_music.py → analyze_srt.py  │
│  → generate_scene_imgs.py → precise_assemble.py           │
│  每个 Skill 都是独立的插件，通过 JSON 契约通信               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   Layer 2: 编排层 (Orchestrator)              │
│              produce_mv.sh - 状态轮询与调度                   │
│  读取 metadata.json → 判断跳过/执行 → 捕获异常 → 熔断       │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                   Layer 1: 底座层 (Infrastructure)             │
│  config.sh (环境变量) + init_project.sh (初始化)             │
│  project_manager.py (状态管理) + llm_logger.py (日志)        │
└─────────────────────────────────────────────────────────────┘
```

## 五、工程收益

| 收益类型 | 具体表现 |
|----------|----------|
| **成本节约** | 断点续传避免重复生成已完成的图片 |
| **效率提升** | 多线程并发生成图片 |
| **可靠性** | 指数退避重试机制处理网络波动 |
| **可维护性** | 模块独立，可单独替换 |
| **可观测性** | 全链路日志追踪 |
| **可迁移性** | 环境配置与业务逻辑分离 |
